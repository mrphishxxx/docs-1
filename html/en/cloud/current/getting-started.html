Getting Started

Sign Up for a Trial and Get Up and Running with Hosted Elasticsearch!

Introduction

This tutorial will walk you though how to sign up for a trial and get started using Elastic Cloud.

What is Included in the Trial?

The free trial includes one cluster with a total of 1GB memory, 16GB storage, and high availability (HA) across two zones.

How many documents? As many as you can fit in that node. This will of course be dependent on document size and what kind of queries you execute. You may also choose any of the official plugins that are listed in the Elastic Cloud Console.

Should you want to enable dynamic scripts, we ask that you either register a credit card on the Elastic Cloud console or visit the support page for help. This is simply because we don’t want to run your code without knowing who you are. More background on this choice is available in this article.

How to Sign Up

The signup process is really simple and straightforward. All that is required is an email address. Credit card is optional for the trial and may be added at any time later if desired. Sign up now. Fill in your email address and click accept for the legal terms. Then go and check your mailbox and click on the link in the email. If you haven’t received the email within a few minutes then remember to check your spam folder.

You will be asked to set a password for your account. Once this is done you will be signed in to the Elastic Cloud Console which looks like this:

Create Your First Cluster

  1. Click on "Clusters" in the top navigation. You can now start to configure your cluster.
  2. Set the capacity (memory/disk space). During the free trial period, you get access to provision one cluster with 1GB memory, 16GB storage, and HA across two zones.
  3. Choose a region close to you. Note: Region cannot be changed after the cluster has been created.
  4. Select level of fault tolerance. The trial includes 2 data centers.
  5. Select Elasticsearch version. The default version is the latest, stable version. Note: you will not be able to downgrade once your cluster is up and running.
  6. We strongly recommend disabling dynamic scripts (default selection). Your other options are to enable sandboxed dynamic scripts or dynamic scripts. Read more about dynamic scripts here: Elasticsearch Security - Scripting for Fun and Profit.
  7. Select number of shards. The default is 1. We recommend that you read Sizing Elasticsearch before your change the number of shards.
  8. Choose whether you want to enable or disable automatic index creation.
  9. Choose to either require explicit name for deletions or allow wildcards.
  10. Optional: select from a list of plugins, e.g. Kibana.
  11. Optional: name your cluster.
  12. Click on the Create Cluster button. Your cluster will be up and running in seconds!

To get back to this page once your cluster is created, simply select "Configuration" in the left sidebar. For more details on the various configuration parameters, see configuration page.

Kibana 4

You get Kibana 4 for free with every cluster you create. In order to use Kibana, your cluster must be running Elasticsearch version 1.6 or newer. To enable Kibana 4, go to the cluster configuration page on the Elastic Cloud console. At the bottom of the page you will see the Kibana 4 pane. Simply click on the Enable button, and within seconds you will see the URL which gives you access to Kibana 4.

Next, you need to provide the Kibana 4 endpoint with credentials so that you can access Elasticsearch resources, e.g. granting read access to relevant Elasticsearch indexes and granting update access to the .kibana index.

Connecting to Your Cluster

After creating the cluster you are redirected to the overview page for your new cluster. Once the instance has status running you can click one of the endpoints and connect to it using your browser. Be aware that it may take a few minutes for the new DNS records to become active. If the overview page lists your instance as running and you’re having issues connecting to the endpoint, give it five minutes and try again. Once your browser connects to the endpoint you should get see a standard welcome message like this:

{

    "status": 200,
    "name": "instance-0000000000",
    "version": {
        "number": "1.0.0",
        "build_hash": "a46900e9c72c0a623d71b54016357d5f94c8ea32",
        "build_timestamp": "2014-02-12T16:56:08Z",
        "build_snapshot": false,
        "lucene_version": "4.6"
    },
    "tagline": "You Know, for Search"

}

There are many ways of connecting to an Elasticsearch cluster and we do our best to support them all. Currently we support:

  • HTTP, used by curl and most non-JVM languages.
  • Java transport client. A good choice if your application runs on the JVM. Requires a plugin.

If this is your first time using Elasticsearch we recommend going with the HTTP protocol and some curl commands to become familiar with the basics. If you’re on a *nix based OS you probably already have curl installed, or can easily install it from your package manager. On the Overview page for your cluster there is a quick-start with a few examples using curl that you may cut and paste into your shell. Once you experiment with slightly bigger queries you might want to try something that is a little more syntax aware. Some popular choices are:

  • Chrome sense plugin
  • The query editor in Kopf
  • Elastic-hammer

Dashboards

We provide access to several Elasticsearch dashboards - hosted directly from our proxy - including Kopf and Kibana.

The Next Step: Adding a Credit Card

To add or change credit card, click your email address in the top right corner, and select billing details in the drop down menu. Your credit card information will then be sent securely to our billing provider and stored with them. Your credit card will not be charged unless you increase the capacity beyond the free trial offer and/or add a third data center, or the trial expires while the cluster is still running. Should you wish to stop the cluster, use the "delete cluster" button at the bottom of the configuration page.

Access Control

Once you either enable dynamic scripts or start adding sensitive data to your cluster we highly recommend that you put sensible security measures in place - security by obscurity won’t do! At Elastic Cloud we have many different customers with different setups and different needs, so the full documentation of ACLs might seem a little daunting at the first read, but with these few quick steps you will get a read only user and one read and write user allowed to connect over HTTP.

  • Click "Access control" available in the left hand menu of the console
  • Scroll down to "Sample Authenticated" and click "Load into ACL-editor"
  • Take note of the password for the two users defined, change them if you like, but don’t put in any password used anywhere else.
  • Click save.
